### 动机：为什么是线程？
- 进程是一个在计算机中非常重量级的要求
	- 它需要占用很大量的资源，并且非常复杂。
- 然而对于每一个动作都单开一个进程，非常不划算。
- 对一些具有复杂功能的程序，如果一直开新的进程，进程数会非常多，且占用、浪费大量计算机资源，不利于计算机的任务。
	- 例如教务服务系统，如果每一个客户的申请都要创建进程，那么进程很快就不够用了。
- 解决的方法：用更轻量级的”进程“
- 线程：共用数据段、代码段、打开的文件，仅需要重新分配寄存器、PC、栈，花费更少资源。
- 好处：
	- 响应能力与实时性
	- 资源共享，节省内存空间
	- 经济性，更好地调用CPU的能力

### 多处理器（核）运行
- 多处理器运行所需要的需求如下：
	- 功能与数据的划分。
	- 平衡，负载均衡。现在执行的并行普遍存在的问题
		- 不同线程和进程难以确定完成的结束时间是否一致。
			- 很多的代码存在分支、循环语句，难以判断。
	- debug testing过程难以把控。
- 针对多核与多处理器情景，现在有两项技术：
	- Parallelism 并行。在物理上并行，有多处理器，实实在在地分开运行。
	- Concurrency 并发。单处理器上处理多任务，交替执行。（multitasking）
	- 现在很多时候两者都是一起使用。
- 并行的类型：
	- 数据并行：将大数据块分成小数据，分到几个核里面分别执行。
	- 任务并行：根据任务的执行顺序将它切分，将可并行的部分并行运行。
- 并行层级：
	- 代码层并行：程序员在编写代码时就设置程序的并行模式。
	- 函数级、任务级并行：OS将不同函数分成不同线程同时执行。
	- 编译并行：编译器将不同的运算拆分开，同时执行。
	- CPU并行：在硬件层面并行。

### 性能定律
- 阿姆达尔定律：性能的提升率受限于提升性能的部分在整个程序中的占比。

### 用户线程与内核线程
- 用户线程：用户级线程库管理的线程。
- 用户级线程怎么和内核级线程对应？
	- 多对一：多个用户级线程对应一个内核级线程
		- 优点：内核开销很小
		- 缺点：服务质量很差，延迟会很高。
		- 很少有系统会采用这种模型。
	- 一对一：一个用户级线程对应一个内核级线程（普遍采用）
		- 缺点：内核开销较大。目前可控。
		- Windows, Linuxs采用的模型
	- 折中态：多对多：多个用户级对多个内核级，当然内核级会少一点。
		- 新的问题：映射特别复杂。操作系统的管理变得很麻烦。
	- 折中态：混合模式，一部分多对多，一部分一对一。
		- 依旧存在管理复杂的问题。

### 三种创建线程的方式：
- Thread pools 线程池
	- 在应用中大量需要多线程。创建一个线程池，应用对线程池进行申请，可以实现多线程资源的按需使用。
	- Application -> Task Queue -> Thread Pool
	- 应用程序将任务提交给cpu的任务队列，再分配到线程池里面的线程。
	- 为什么要使用线程池？
		- 性能：通过线程池回收、分配来保持线程数量的恒定，不需要反复创建销毁来消耗额外性能。
		- 扩展性
		- 资源管理效率很高，按需使用。
	- Java线程池实例。
- Fork-Join联合操作：
	- main thread -> fork 变为子任务 -> join 合并回主线程
	- 递归操作
- OpenMP：shared memory 模式的进程通信
	- 为多个线程、进程创建共享内存区域。
	- `# pragma omp parallel`
		- 先导入`#include <omp.h>
		- 声明openMP并行区域`# pragma omp parallel{}`，花括号内的代码都共享内存并且并行执行。
		- 设置线程数量`export OMP_NUM_THREADS=5`
		- 编译`gcc -o hello -fopenmp hello.c`
		- 执行`./hello`

### 线程的问题
- 1. 线程的`fork() join() exec()`会和进程同名命令产生冲突。
	- UNIX 进程：
		- `clone() = UNIX fork()`，两者同样。
	- Linux thread = LWP
		- `clone()`->线程，`fork()`->进程
- 2. 线程的通讯方式只有signal，不如进程的通讯方式多样
	- 同步或异步
	- signal
		- 信号包括系统自带的信号和用户自定义的信号
		- 信号的收发考虑的地方非常多。
- 3. 线程的取消非常复杂。线程的取消会对操作系统的调度、资源管理产生巨大的影响和问题。
	- 异步取消：当前的线程马上取消
	- 延期取消：多线程运行过程中周期性取消。
	- 线程取消后：
	- Java中调用 对象`.interrupt()`进行取消
- 4. 多线程对本地存储带来挑战，会对内存与硬盘的性能产生挑战。
- 5. 多线程的调度问题非常显著，需要非常优秀的算法。

### 线程实例
- windows：
	- 用户与内核线程一一对应
		- 线程有三类：
			- ETHREAD：包含进程信息核指向KTHREAD的指针
			- KTHREAD：内核线程
			- TEB：用户线程
- linux：
	- 更倾向于用task而非线程。
	- 使用`clone()`创建。允许子线程共享父进程的所有内存空间。

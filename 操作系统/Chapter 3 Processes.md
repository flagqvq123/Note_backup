### Process 进程 - ==正在执行的程序==
	进程的定义：正在执行中的程序
一个进程在内存中被分配的空间可以分为多个区域：
- 代码段
- 数据段：储存全局参数
- 栈(Stack)：储存函数调用时的参数等
- 堆(Heap)：程序运行时动态分配的内存，储存局部变量
	- 堆栈都会动态变化，并且相互增长的过程靠近，但是操作系统必须保证它们不会重叠。
	- 代码段与数据段在运行过程中不会变化

不同的独立进程在内存中保持相对独立的地位，即使是同一程序的副本，它们的关系也仅仅是代码段里面的数据相同，实际上仍然被分配了不同的内存地址。

### Process State
- 一个进程有五种状态：
	- New：正在创建。
	- Running：正在执行。
	- Waiting：正在等待某些事件发生。
	- Ready：正在CPU中分时进行。
	- Terminated：进程执行完毕，终止。
- 这五个状态贯穿了整个进程的生命周期。

##### Process Control Block (PCB) 进程控制块
- 用于存储每个进程的信息：作为==启动或重启==一个过程所需的所有数据的存储库，同时包含一些统计信息。
	- Process state
	- Program counter , CPU registers 在发生中断时需要存储，保证随时恢复并正确运行
	- pid：十六位二进制数，表示进程的唯一编号。也就是说操作系统最多负载65535个进程数。
	- 调度信息：该信息包括先前的进程、调度队列指针以及任何其他调度参数。
	- 内存管理信息
	- 父子进程关系信息（链表形式）
	- IO状态信息、统计信息...
- task_struct 结构体非常长，有几百行代码。

##### Threads
- 线程是进程的运行者。上述的例子都简化成为了一个进程仅有一个线程的情况，然而实际情况是可能一个进程包含多个线程同时执行，从而允许同时执行多个操作。
- 这个功能在多核系统中特别有用，因为多个线程可以并行运行。
### Process Scheduling
 进程控制器从可用进程中选择下一个在CPU核上执行。而这个选择是从队列中选择：
- Ready queue：可以执行的进程。
- Wait queue：等待中的进程。
	- 进程在不同队列中迁移。

### Context Switch(上下文切换)
- 在切换进程时候，需要保存老进程信息，同时加载新进程信息。
- 这个过程被称为上下文切换。
- 上下文切换开销在百微秒级别，这个开销实际上很高。在设计切换时需要考虑到切换的开销。

### Operations on Processes
##### Process Creation
- 进程以树的形式构成，一个进程可能会创造多个子进程，从而构成树形结构。
- 每个进程拥有独立的pid来区分。

>[!tip]-
>ps command : 查看正在运行的进程
>```
PID    进程编号
TTY    类型
STAT   进程状态
START  开始运行的运行时间
>```
pgreg [process name] 搜索一个进程名，返回pid

- 进程创建总是由父进程创建子进程。
	- 也可选择只继承父进程的部分资源、或是完全不共享资源。
	- 强制让子进程继承父进程的资源可以有效防止过多的进程创建使得系统资源过载。
	- 除了提供各种物理和逻辑资源外，父进程还可以将初始化数据 (输入) 传递给子进程。
- 父进程创建新的子进程后：
	- 父进程可能与子进程同时并行执行，也可能等待直到子进程结束。
	- 子进程可能是父进程的副本，拥有相同的代码与数据，也有可能是一个新的加载的程序。
- 地址空间：子进程通常继承父进程的地址空间。
	- Unix examples：
		- `fork()`：创建子进程，会克隆所有的信息。
		- `exec()`：执行进程的任务
		- `exit()`：关闭子进程
		- `wait()`：父进程继续执行

##### C program forking separate process
- `pid = fork()`创建子进程
- 对Unix中，进行`fork()`操作后，父进程与子进程分别独立继续运行代码，唯一的区别是父进程中`fork()`会返回子进程的pid值，而子进程的`fork()`返回0。通过这种方式区分子、父进程。
- 一般来说，子进程会在生成后执行`exec()`系统调用，将整个程序跳转到新的程序上。`exec()`系统调用会加载目标可执行程序到该进程，销毁原来进程的所有内存（包括代码），然后执行该新程序。
	- 因为对`exec()`的调用会用一个新程序覆盖进程的地址空间，所以`exec()`不会返回控制，除非发生错误。
- 通过这种方式，两个进程能够进行通信，然后各自独立运行。父进程随后可以创建更多子进程;或者，如果在子进程运行期间没有其他事情可做，它可以发出 wait() 系统调用，将自己从就绪队列中移出，直到子进程终止。

### Process Termination
- 进程执行结束后使用`exit()`结束。将状态数据从子进程返回给父进程 (通过`wait()`)
- 如果父进程被终止，那么子进程将会被同时终止。

### 生产者-消费者问题
- 对于协作进程来说，生产者进程生产数据存在内存中，而消费者进程负责消耗数据。
- 对于共享的内存空间，倘若两个进程在交错进行时同时访问内存，则可能会导致该内存空间错乱。


### Pipes 通信管线
- Ordinary Pipes 日常管道：普通传输通道，带宽有限。
	- 父进程创建一个子进程，原先的数据变为父子进程通过管道同步共享。
- Named Pipes 命名管道：带宽更大。
- 管道调度算法：
	- FIFO：先进先出。（等同于FCFS）
- 管道命令：
	- `mkfifo pipe1`使用**mkfifo**创建FIFO管道。
- 实现管道的代码：
	- c语言中用mknod创建管道。

### Sockets 套接字
- `socket`被定义为网络上的终端。在客户端与服务端分别创建一个socket进程，在获得两边的IP地址和端口号后，利用TCP的技术传输。
	- IP：internet protocol 互联网协议，其定义互联网上每一个地址。
	- TCP：Transmission Control Protocol ： 传输控制协议，这是一个传输层协议。它 的传输模式是点对点的传输。
		- 传输细节：
			- 三次握手同步。
			- 同步完成后开始数据传输。
			- 数据丢失会重新传输。
		- 相对稳定可靠。
		- 缺点：一对多、多对多情况下不高效。
	- UDP：User Datagram Protocol： 用户数据电报协议。牺牲可靠性，用广播、多播的方式对一对多的情境下进行传输。

### MPI：Message Passing Interface
- 消息传递接口是一个标准化与可移植的函数库用于并行计算，特别是高性能计算环境。
- 在80年代末到90年代初，慢慢形成了 MPI 的框架。
- MPI 的影响非常广泛。
- 常见的MPI通信：
	- broadcast：广播。
	- scatter：将数据分块切分传输。
	- gather：聚合。
	- reduction：（最常见）将任务分散后通过gather聚合，再通过计算等获得最终结果。
- 程序的运行：
	- 导入：MPI include file
	- 声明运行，入口：Initialize MPI environment
	- 声明结束，出口：Terminate MPI environment

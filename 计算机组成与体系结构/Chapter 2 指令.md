## 2.3 计算机的操作数

##### 2.3.1 存储器操作数
	- 复杂存储结构保存在内存中。内存本质是一个超大数组，访问需要提供地址下标。
	- CPU的算数运算只作用于寄存器，因此RISC-V包含在内存和寄存器之间传输数据的指令，称为**数据传输指令**

- 载入指令： 将数据从内存复制到寄存器的指令`(load)`
	- 格式是 `操作指令 目标寄存器1 偏移常量（基址寄存器2）`，表示将地址为（基址+偏移量）的内存地址上的数据传入到寄存器1中。
	- 实际 RISC-V 指令名称为 `lw` 表示取字
>[!tip] 字与字节
>字节寻址会影响数组下标。几乎所有体系结构都是按单个字节寻址的，因此字的地址于字内的四个字节之首的地址是相匹配的，并且连续字的地址相差4。

> [!tip]+ 例子：
> `ld   x9, 32(x22)    // Temporary reg x9 gets x22[8]`
> ---
> 这个指令将位于内存`x22[8]`的数据存到`x9`寄存器中

- 存储指令：与载入指令相反的指令，它从寄存器复制数据到内存。
	- 存储指令的格式类似于载入指令的格式：`操作名称 目标寄存器 基址寄存器 偏移量`。
	- 实际 RISC-V 指令名称是 `sw` 表示存字
>[!tip]+ 例子：
>	假设变量h存放在寄存器 x21 中，数组 A 的基址存放在寄存器 x22 中。C赋值语句的 RISC-V 汇编代码是什么？
>		`A[12] = h + A[8]`
>---
> >[!tip]- 答案：
> >	```
> >		lw    x9, 32(x22)
> >		add   x9, x21, x9
> >		sw    x9, 48(x22)
> >	```

##### 2.3.2 常数或立即数操作数
		含有常数的指令非常常见。然而仅仅使用上面的指令我们需要将常数从内存载入到寄存器，再进行使用。接下来将介绍另一个版本的算数指令来避免这个问题。

- `addi` ： 立即数加。它的一个操作数是常数。如果需要将 4 加到寄存器 x22 中，只需写成：
```
	addi   x22, x22, 4    // x22 = x22 + 4
```
- 常数 0 有很频繁的作用，例如可以使用常 0 寄存器求原数的相反数。因此，RISC-V 专用寄存器 `x0` 硬连线到常数 0。

### 2.4 有符号数与无符号数
- 最低/高有效位：最右边（最小）是**最低有效位**（LSB, least significant bit），最左边是最高有效位（MSB）
- 计算机需要区分正负数，所以使用最高位表示一个负数。约定使用==*二进制补码*==表示法。
>[!tip]+ 二进制转换为十进制:
>下面这个32位二进制补码的十进制数是多少？
>	`11111111 11111111 11111111 11111100`
>---
>>[!tip]- 答案：
>>将数字的位值代入上述公式：
>>```
>>		(1*-2^31) + (1*2^30) + (1*2^29) + ... + 1*2^2 + 0 + 0
>>		= -2^31 + 2^30 + ... + 2^2 + 0 + 0
>>		= -4
>>```
>>捷径：
>>```
>>		将所有位的数反转再加1，就可以得到该数字的绝对值。
>>		同理，如果要从正数变为负数，就可以先减1，再将所有位的数字反转。
>>```

- 二进制补码的优势是所有负数的最高有效位都是 1 ，因此硬件只需要检测这位就可以查看是正数还是负数。（数字 0 被认为是正数）
- 二进制补码表示确实有一个负数而没有相应的正数：`-2147483648`。即便如此，现在所有的计算机都是用二进制补码来表示有符号数。
- "溢出" ： 当数字超出容量就会产生溢出，此时最左边的位与表示逻辑相反。

### 2.5 计算机中的指令表示
	机器指令和人写出的指令不一样。本节将解释这种差异
- 指令以一系列高低电平表示的 0，1 信号储存在计算机中。实际上，每条指令的各个部分都可以被视为一个单独的数。

##### RISC-V 字段
按顺序给 RISC-V 字段命名，使其更易于讨论。

##### R型指令格式(Register)

| funct7 | rs2  | rs1  | funct3 | rd   | opcode |
| ------ | ---- | ---- | ------ | ---- | ------ |
| 7bit   | 5bit | 5bit | 3bit   | 5bit | 7bit   |
- opcode（操作码）：指令的基本操作。
- rd：目的操作数寄存器，存放操作结果
- funct3：一个另外的操作码字段
- rs1：第一个源操作数寄存器
- rs2：第二个源操作数寄存器
- funct7：一个另外的操作码字段
`这种指令需要适配类似“载入”指令的时候，需要立即数占用一个位置。此时rs、rd最多5位，只能表示最大31的常数。所以我们需要使用另一种指令格式。`

##### I型指令格式(Immediate)

| immediate | rs1  | funct3 |  rd  | opcode |
|:---------:|:----:|:------:|:----:|:------:|
|   12bit   | 5bit |  3bit  | 5bit |  7bit  |
- immediate：补码，表示-2^11 到2^11 - 1之间的整数

##### S型指令格式


| immediate[11:5] | rs2  | rs1  | funct3 | immediate[4:0] | opcode |
| :-------------: | :--: | :--: | ------ | :------------: | :----: |
|      7bit       | 5bit | 5bit | 3bit   |      5bit      |  7bit  |
- 同样地，rs2、rs1代表两个源操作数寄存器。
- immediate被拆分成两个地方分别存储，一个在原来的funct7，一个在rd。

>[!tips] rs, rd 的英文含义
>rs : register sourse ; rd : register destination

**RISC-V体系结构设计师选择这种设计是因为它能够在所有指令格式中保持`rs1`和`rs2`字段在==相同的位置==。** 保持尽可能相似的指令格式降低了硬件的复杂性。同样，`opcode`和`funct3`字段也总是保持同样的大小、在同一个位置。

机器通过读取操作码(`opcode`)字段来区分，每个格式在第一个字段中被分配一组不同的操作码值来让硬件知道如何处理剩余部分。

>[!tip]+ 汇编器
>在处理常量时，RISC-V汇编语言不会强迫程序员使用addi，而是会根据操作数来自动生成正确的操作码和指令格式。不过为了方便，我们在后续的讲解中都会使用显式名称。

>[!tip]+ 寄存器数量
>保持所有指令长度相同会使得寄存器数量收到限制。鉴于此和“更少更快”的原则，现在的大多数指令系统体系结构都是设置16或32个通用寄存器。

>[!tip]+ “存储程序”
>当前的计算机构建基于两个原则：
>1.指令由数字形式表示。
>2.程序和数据一样保存在存储器中来进行读写。
>这些原则引出“存储程序“的概念，数据与程序都使用数字形式存储在存储器中，这意味着存储器可以理论上有同时承担程序和数据的能力。存储器可以存储一个编辑器程序的源代码。
>采用相同的处理方式极大简化了存储器硬件和计算机系统软件。

### 2.6 逻辑操作
	对一个字内部的位或者几个位构成的字段进行操作的指令被称为逻辑操作。

| 逻辑操作名 | C操作符 | RISC-V指令  |
| :---: | ---- | --------- |
|  左移   | <<   | sll, slli |
|  右移   | >>   | srl, srli |
| 算数右移  | >>   | sra, srai |
|  按位与  | &    | and, andi |
|  按位或  | \|   | or, ori   |
| 按位异或  | ^    | xor, xori |
| 按位取反  | ~    | xori      |
	tips: 实现NOT的方式是使用XOR并且另一个操作数是全1

- slli 与 srli ：左移逻辑立即数，右移逻辑立即数。将某个寄存器中的操作数左/右移常数个位，存入指定的寄存器。
```
slli  x11, x19, 4 // reg x11 = reg x19 << 4 bits
```
	- 这两个移位指令使用I型格式。但是因为一个寄存器中存储最多32位，所以I型格式中的12位immediate字段中只有最后5位被使用，其他的位被重用作额外的操作码字段，即funct7.
	- 逻辑左移i位等效于乘以2^i，就像类比十进制一样。
- srai ：算数右移立即数。与srl区别在于它的右移会保持符号位不移动。
- srl，sra，srl ：它从寄存器中取出移位的位数而非立即数。

- and ：按位与操作。只有两个操作数的某一位都是1时，结果的对应位才是1。
	- 由于and可以在源操作数的某些位是0时，让结果数的对应位设为0，所以这种与and联合使用的操作数被成为掩码。
```
 // x11 = 00000000 00000000 00001101 11000000
 // x10 = 00000000 00000000 00111100 00000000
 
 and  x9, x10, x11  // reg x9 = reg x10 & reg x11
 
 // x9 =  00000000 00000000 00001100 00000000
```
- or : 按位或操作。两个操作数的某一位存在一个是1时，结果的对应位就是1.
- not -> xor ：为了保持三操作数格式，RISC-V的设计者引入指令xor（异或）来代替not。not等价于异或1111....111
- RISC-V还提供了andi、ori、xori的立即数指令。
